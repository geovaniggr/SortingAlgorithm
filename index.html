Algoritmos de Ordenação

O presente estudo tem como objetivo mostrar a complexidade assintótica de algoritmos de ordenação, para isso foi escolhido dois algorítmos diferentes: QuickSort e Bubblesort.

Como foi feito?

Para gerar os gráficos utilizei como espaço amostral 30 arrays com tamanhos diferentes

  • 10 arrays: 1.000 elementos
  • 10 arrays: 10.000 elementos
  • 10 arrays: 100.000 elementos

Materiais Utilizados:

  • Javascript
  • Node.JS
    • *Durante a execução dos testes não houveram atividades paralelas no computador.

BubbleSort

Esse algorítmo tem como característica comparar valores em posições consecutivas, na primeira chamada do método é escolhido o elemento na posição inicial como pivô, para uma ordenação crescente será feito a seguinte comparação:

  • Pivô > Elemento Adjacente: Troque de Posição
  • Pivô < Elemento Adjacente: Mantenha a posição e o pivô se torna o elemento adjacente
Após isso será necessário percorrer o array novamente, entretanto podemos percorrer 1 elemento a menos pois a última posição contém o maior valor do array.

QuickSort

O quicksort também tem como característica ter um pivô para comparação de valores, entretanto o algoritmo utiliza o paradigma de Divisão e Conquista, reduzindo o problema recursivamente, na primeira chamada o array pode ter como pivô um valor em uma posição aleatória, e a partir disso é feito comparações com os elementos para gerar a seguinte disposição:

  • Valores menores à esquerda do pivô
  • Valores maiores à direita do pivô
Após isso o array estara ordenado em relação ao pivô, basta agora utilizar o quicksort à direita e à esquerda recursivamente para que ao desempilhar as chamadas ter um array ordenado.
