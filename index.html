Algoritmos de Ordenação

O presente estudo tem como objetivo mostrar a complexidade assintótica de algoritmos de ordenação, para isso foi escolhido dois algorítmos diferentes: QuickSort e Bubblesort.

Como foi feito?

Para gerar os gráficos utilizei como espaço amostral 150 arrays com tamanhos diferentes

  • 50 arrays: 1.000 elementos
  • 50 arrays: 10.000 elementos
  • 50 arrays: 100.000 elementos

Materiais Utilizados:

  • Javascript
  • Node.JS
    • *Durante a execução dos testes não houveram atividades paralelas no computador.

BubbleSort

Esse algorítmo tem como característica comparar valores em posições consecutivas, na primeira chamada do método é escolhido o elemento na posição inicial como pivô, para uma ordenação crescente será feito a seguinte comparação:

  • Pivô > Elemento Adjacente: Troque de Posição
  • Pivô < Elemento Adjacente: Mantenha a posição e o pivô se torna o elemento adjacente
Após isso será necessário percorrer o array novamente, entretanto podemos percorrer 1 elemento a menos pois a última posição contém o maior valor do array.

QuickSort

O quicksort também tem como característica ter um pivô para comparação de valores, entretanto o algoritmo utiliza o paradigma de Divisão e Conquista, reduzindo o problema recursivamente, na primeira chamada o array pode ter como pivô um valor em uma posição aleatória, e a partir disso é feito comparações com os elementos para gerar a seguinte disposição:

  • Valores menores à esquerda do pivô
  • Valores maiores à direita do pivô
Após isso o array estara ordenado em relação ao pivô, basta agora utilizar o quicksort à direita e à esquerda recursivamente para que ao desempilhar as chamadas ter um array ordenado.

Como é possível perceber o pivoteio do Quicksort se torna muito mais eficaz do que as trocas constantes do Bubblesort, ainda que no pior caso o Quicksort possa ser quadrático, comprovando que:

  • Quicksort ∈ Θ(n.logn)
  • BubbleSort ∈ O(n²)

MergeSort x Quicksort

Como a diferença no tempo de execução entre o Quicksort e o BubbleSort são de fato gigantes, optei por ver como se comporta um outro algoritmo que ∈ O(n * logn), no caso o Mergesort, esse também utiliza o método da divisão e conquista, entretanto, a estratégia é diferente, o Mergesort divide o array principal em 2 e recursivamente quebra essas partes até o tamanho minimo de 2 elementos e ao desempilhar ordena os elementos corretamente, tornando-se apenas necessário intercalar os 2 arrays ordenados no final.

MergeSort

MergeSort x QuickSort

Como é possível observar os tempos de ambos algoritmos são bem semelhantes, entretanto, o MergeSort demonstra ter uma grande variação de tempo em relação a disposição de elementos no array.

A diferença entre ambos algoritmos está na constante absorvida na análise assintótica, onde o quicksort leva vantagem, ocasionando que seu tempo de ordenação na pratica seja muito mais eficiente. Como apresentado, existe um caso onde o Quicksort demonstra ter problemas: arrays ordenados.

Para um último teste foi tomado um array ordenado de tamanho 100.000

  • Mergesort demorou: 0.228 segundos para fazer a ordenação
  • Quicksort não conseguiu fazer a ordenação.
O metodo particiona encontrou dificuldades para conseguir ordenar um array ordenado, isso porque a cada execução será necessário empilhar um array de (n-1) elementos de forma que o Node.js, rapidamente antingiu o tamanho máximo de pilhas de chamadas.
O Quicksort conseguiu ordenar no máximo 8300 elementos de um array e demorou 0.211 segundos, demonstrando que para esse caso em específico sua complexidade é realmente quadrática.
